# ロールプレイングプロンプト（役割定義）

## 概要

ロールプレイングプロンプトは、AIに特定の専門家や役割を演じさせることで、その分野に特化した質の高い回答を引き出す手法です。AIに「あなたは〜の専門家です」と役割を明示的に与えることで、より専門的で文脈に沿った回答が得られます。

## 重要度

**⭐ 必須** - すべてのプロンプトで最初に定義すべき基本手法

## 使用場面

- 専門的な知識や視点が必要な質問をする時
- 特定の文脈や立場からの回答が欲しい時
- 一貫性のあるトーンや文体で回答してほしい時
- 複数の専門分野にまたがる複雑な課題を扱う時
- 特定の業界や職種の実務的なアドバイスが必要な時

## 基本パターン

```
あなたは[専門分野]の[役職/専門家]です。
[経験年数や特徴的なスキル]を持っています。
[具体的なタスクや質問]
```

### バリエーション

**シンプル版:**
```
あなたは〜の専門家として回答してください。
```

**詳細版:**
```
あなたは以下の役割を持つ専門家です：
- 専門分野: [分野名]
- 経験: [年数や実績]
- 得意領域: [特定のスキルや知識]
- 視点: [どのような立場で考えるか]

上記の役割に基づいて、[タスク]を実行してください。
```

## 実務例

### 例1: マーケティング戦略の立案

**プロンプト:**
```
あなたは10年以上の経験を持つデジタルマーケティングのシニアストラテジストです。
特にSNSマーケティングとコンテンツマーケティングに精通しており、
複数のスタートアップを成功に導いた実績があります。

クライアント情報:
- 業種: オンライン英会話サービス
- ターゲット: 30-40代のビジネスパーソン
- 予算: 月額50万円
- 目標: 3ヶ月で無料体験登録者を1000人獲得

上記のクライアント向けに、SNSマーケティング戦略を提案してください。
```

**期待される出力の特徴:**
- マーケティング用語を適切に使用
- 予算配分の具体的な提案
- KPIの設定と測定方法
- 実務的なタイムラインとアクションプラン
- リスクと対策も含めた実践的な内容

### 例2: コードレビュー

**プロンプト:**
```
あなたはシニアソフトウェアエンジニアで、特にPythonとクリーンコードの原則に精通しています。
15年以上の開発経験があり、コードレビューにおいては以下の観点を重視します：
- 可読性と保守性
- パフォーマンス
- セキュリティ
- ベストプラクティスへの準拠

以下のPythonコードをレビューし、改善提案をしてください。

```python
def calc(a,b,c):
    result=a+b*c
    return result
```
```

**期待される出力の特徴:**
- 命名規則の改善提案
- ドキュメンテーションの追加
- 型ヒントの提案
- エッジケースの考慮
- より良い実装例の提示

### 例3: 医療アドバイス（一般的な健康情報）

**プロンプト:**
```
あなたは10年以上の臨床経験を持つ内科医です。
患者にわかりやすく医学的な情報を説明することが得意で、
エビデンスに基づいた情報提供を心がけています。

※注意: あくまで一般的な健康情報として回答し、個別の診断や治療は推奨しないでください。

質問: 睡眠の質を改善するための科学的に裏付けられた方法を教えてください。
```

**期待される出力の特徴:**
- 医学的根拠に基づいた説明
- 専門用語を適度に使いつつ、わかりやすい解説
- 複数のアプローチを提示
- 注意事項や個別相談の必要性にも言及

### 例4: UXデザインの評価

**プロンプト:**
```
あなたはUX/UIデザインの専門家で、特にモバイルアプリのユーザビリティ評価に精通しています。
Google、Appleのデザインガイドラインを熟知しており、
アクセシビリティとユーザー中心設計を重視します。

以下のモバイルアプリの登録フローを評価し、改善提案をしてください：

1. メールアドレス入力
2. パスワード設定（8文字以上）
3. 氏名入力（姓・名）
4. 生年月日選択
5. 住所入力（郵便番号、都道府県、市区町村、番地）
6. 電話番号入力
7. 利用規約同意
8. 登録完了
```

**期待される出力の特徴:**
- UX原則に基づいた評価
- ステップ数の最適化提案
- 入力負荷の軽減方法
- エラー防止とガイダンスの改善案
- A/Bテストの提案

## 組み合わせ

### 推奨される組み合わせ

**Role-playing + CoT（思考の連鎖）:**
```
あなたはベテランの財務アナリストです。
以下の企業の財務諸表を分析し、投資判断を下してください。
分析プロセスを段階的に説明しながら結論を導いてください。

[財務データ]
```

**Role-playing + Zero-shot CoT:**
```
あなたは経験豊富なプロジェクトマネージャーです。
以下のプロジェクトリスクについて、step by stepで対策を考えてください。

[リスクシナリオ]
```

**Role-playing + 出力形式指定:**
```
あなたはテクニカルライターです。
以下の技術仕様を、初心者向けのMarkdown形式のドキュメントにまとめてください。
見出し、箇条書き、コードブロックを適切に使用してください。

[技術仕様]
```

**Role-playing + A to Z:**
```
あなたは起業支援の専門家です。
飲食店開業に必要なすべての手順を、法的手続きから資金調達、
マーケティングまで包括的にリストアップしてください。
```

## 注意点

### ✅ 効果的な使い方

1. **具体的な役割定義**
   - ❌ 「あなたは専門家です」
   - ✅ 「あなたは10年以上の経験を持つSEO専門家で、特にテクニカルSEOに精通しています」

2. **文脈と制約の明示**
   - 経験年数、得意分野、重視する観点を明確に
   - 回答のトーンや立場も指定すると効果的

3. **複数の役割を組み合わせる**
   - 「マーケターとデータアナリストの両方の視点から分析してください」
   - 多角的な視点が得られる

### ⚠️ 避けるべきこと

1. **過度に複雑な役割設定**
   - 矛盾する複数の役割を同時に与えない
   - シンプルで明確な役割定義を心がける

2. **不適切な権威付け**
   - 医療診断、法律相談など、専門家の資格が必要な領域では慎重に
   - 「一般的な情報として」など、適切な注意書きを追加

3. **役割の変更**
   - 同じ会話内で頻繁に役割を変えると一貫性が失われる
   - 新しい役割が必要な場合は明示的に切り替える

## ベストプラクティス

### 役割定義のテンプレート

```
あなたは以下のプロフィールを持つ専門家です：

【役職/職種】
[具体的な役職名]

【専門分野】
[メインとなる専門領域]

【経験・実績】
- [経験年数や主な実績]
- [特筆すべきスキルや知識]

【アプローチ】
- [どのような視点や方法論を重視するか]
- [どのような価値観で判断するか]

【制約条件】
- [回答時に留意すべき点]
- [避けるべき内容]

上記の役割に基づいて、以下のタスクを実行してください：
[具体的なタスク内容]
```

### 効果測定

役割定義の効果を確認するポイント:
- ✅ 専門用語が適切に使用されているか
- ✅ その分野の実務的な視点が反映されているか
- ✅ 回答の深さと具体性が向上しているか
- ✅ 一貫したトーンとスタイルが維持されているか

## まとめ

ロールプレイングプロンプトは、AIとの対話における最も基本的かつ重要な手法です。適切な役割定義により、回答の質、専門性、一貫性が劇的に向上します。他のプロンプト手法と組み合わせることで、さらに強力な効果を発揮します。
