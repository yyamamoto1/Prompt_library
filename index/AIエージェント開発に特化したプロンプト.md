# AIエージェント開発要件ヒアリング & 工数見積プロンプト

以下の情報をもとに、AIエージェント開発プロジェクトにおける要件定義、基本設計、詳細設計の各工程において、  
**必要な作業内容と工数の目安**を明確にしてください。

---

## プロジェクト概要（任意入力）

- エージェントの目的・ゴール：
  - （例）ユーザーの代わりに特定システムの操作を自律的に行う
- エージェントのペルソナ・役割：
  - （例）経験豊富なソフトウェアエンジニア、フレンドリーなカスタマーサポート担当
- 対象プラットフォーム：
  - CLI / Web UI / API連携 / チャットボットなど
- 想定ユーザー：
  - （例）社内開発者、一般顧客

---

## 技術スタック

- LLMモデル：
  - （例）GPT-4 / Claude 3 / Gemini / Llama 3 / 自社モデルなど
- エージェント用フレームワーク：
  - （例）LangChain / LlamaIndex / CrewAI / Autogen / 使わない（自社開発）など
- ユーザーインターフェース（UI）：
  - （例）React / Vue / Next.js / Gradio / Streamlit など
- 外部ツール連携（Tools）：
  - （例）各種API / データベース / ファイル操作 / Web検索など
- 記憶（Memory）：
  - （例）ベクトルデータベース（Pinecone, Chroma） / RDB / KVSなど
- インフラ：
  - （例）AWS / GCP / Vercel / Docker / Firebase など

---

## 要件定義

以下を洗い出して一覧化してください：

- エージェントが達成すべき主要なタスクやゴール
- エージェントが利用できるツール（API、関数、データベースなど）の一覧
- 主要な対話シナリオ / ユーザーストーリー
- 禁止事項・制約（やってはいけないこと、アクセスしてはいけない情報など）
- 非機能要件（応答速度、精度、セキュリティ、倫理的配慮など）

---

## 基本設計

- エージェントのアーキテクチャ図（思考サイクル、ツール利用、記憶のフロー）
- プロンプト設計（システムプロンプト、タスク指示プロンプトの基本方針）
- 記憶の設計（短期記憶・長期記憶の使い分け、何を記憶させるか）
- ツール連携の基本設計（各ツールの入力と出力、認証方法）
- 評価方針（タスク成功の定義、評価指標）

---

## 詳細設計

- プロンプトテンプレートの詳細設計（変数、Few-shot例など）
- 各ツールの詳細仕様（APIリクエスト・レスポンス、エラーハンドリング）
- 思考プロセスの詳細設計（ReAct, CoTなどの具体的な思考ステップ）
- エラーハンドリングとフォールバック戦略（LLMが応答不能な場合など）
- 評価・テスト設計（評価データセット、テストシナリオ、自動評価スクリプト）
