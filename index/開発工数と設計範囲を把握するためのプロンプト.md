# 開発要件ヒアリング & 工数見積プロンプト

以下の情報をもとに、開発プロジェクトにおける要件定義、基本設計、詳細設計の各工程において、  
**フロントエンドとバックエンドそれぞれに必要な作業内容と工数の目安**を明確にしてください。

---

## プロジェクト概要（任意入力）

- サービスの目的・概要：
  - （例）ユーザーが会員登録して、商品を購入できるECプラットフォーム
- 対象プラットフォーム：
  - Web / モバイル / 両方
- 想定ユーザー数：
  - （例）初期 1,000人、将来的に10,000人以上

---

## 技術スタック

- フロントエンド：
  - （例）React / Vue / Next.js / TypeScript など
- バックエンド：
  - （例）Django / Node.js / Laravel / Rails / FastAPI など
- インフラ：
  - （例）AWS / GCP / Vercel / Docker / Firebase など
- API：
  - REST / GraphQL / gRPC など

---

## 要件定義

以下を洗い出して一覧化してください：

- システムで提供すべき主要機能のリスト（ユーザー視点と管理者視点で分ける）
- 画面構成（必要なページ、UI遷移）
- 業務フロー / ユーザーフロー
- 外部連携（決済、メール、認証など）
- 非機能要件（セキュリティ、スケーラビリティ、パフォーマンス など）

---

## 基本設計

- 各機能に対する「フロントエンド」「バックエンド」の責務分担表を作成
- 画面単位でのUI構成と機能概要
- API設計概要（エンドポイント一覧、HTTPメソッド、レスポンス例）
- データベース構成（ER図 or テーブル一覧）
- 外部ライブラリ / フレームワーク選定理由

---

## 詳細設計

- 各画面のUI詳細、バリデーション仕様
- コンポーネント単位の設計（React等の場合）
- APIレスポンス設計（スキーマ・ステータスコード設計）
- エラーハンドリング・認可・セッション設計
- モデル定義（フィールド、型、リレーションなど）
- テスト設計（単体 / 結合 / E2E）

---
