**商品タイプ別の納品自動化**

```
【① デジタル商品（PDF/動画/テンプレート）】

納品方法：
・決済完了 → ダウンロードリンク自動送信
・または専用ページにリダイレクト

ツール例：
・Gumroad：購入後即ダウンロード
・Stripe + SendOwl：自動納品
・Google Drive + Zapier：リンク自動送信

フロー：
購入 → Stripe → Zapier → メール（DLリンク付き）

---

【② オンライン講座/会員サイト】

納品方法：
・決済完了 → 会員サイトへ自動招待
・ログイン情報を自動送信

ツール例：
・Teachable：購入と同時にアクセス権付与
・Kajabi：ワンストップで完結
・WordPress + WishList Member

フロー：
購入 → 決済システム → 会員サイトに自動登録
    → ログイン情報メール送信

---

【③ サブスクリプション】

納品方法：
・初回決済完了 → アクセス権付与
・毎月自動課金 → アクセス継続
・解約 → アクセス権自動削除

ツール例：
・Stripe Billing
・PayPal定期支払い
・各種サブスク管理ツール

フロー：
初回購入 → 会員登録 → 毎月自動課金
        → 課金失敗時は自動リマインド
```

**自動化ツール連携の設定**

```
【Zapierでの自動化例】

■ Stripe × Gmail
トリガー：Stripeで支払い完了
アクション：Gmailで購入確認メール送信

■ Stripe × Google Drive × Gmail
トリガー：Stripeで支払い完了
アクション1：Google Driveの共有リンクを取得
アクション2：共有リンク付きメールを送信

■ Stripe × Teachable
トリガー：Stripeで支払い完了
アクション：Teachableにユーザー登録

■ Stripe × Slack
トリガー：Stripeで支払い完了
アクション：Slackに通知（売上確認用）

【設定のポイント】
・テスト購入で動作確認必須
・エラー時の通知設定
・定期的な動作チェック
```

**よくあるトラブルと対策**

```
【トラブルと対策】

① メールが届かない
原因：迷惑メールフォルダ
対策：確認メールに「迷惑メールを確認」と記載
      専用ドメインからの送信

② ダウンロードできない
原因：リンク切れ、容量制限
対策：複数の配布方法を用意
      サポート連絡先を明記

③ 二重決済
原因：重複クリック
対策：決済後のリダイレクト設定
      重複チェック機能の利用

④ 返金対応
原因：顧客都合、商品不良
対策：返金ポリシーの明記
      Stripe/PayPalの返金機能活用

【自動化の限界】
・複雑な問い合わせ → 手動対応
・特殊なケース → 個別対応
・高額商品 → 一部手動確認も検討
```
