# モバイルLP最適化

あなたはモバイルUX/UIの専門家です。

以下の情報をもとに、**スマートフォンで高CVRを実現するLP最適化戦略**を立ててください。

【目的】
- スマホでの閲覧体験向上
- モバイルCVR改善
- 離脱率の低減

【入力】
- 現在のモバイル比率：[例：70%]
- モバイルCVR：[例：1.5%]
- PC CVR：[例：3%]
- LP URL：[任意]
- 主な課題：[例：読み込みが遅い、CTAが押しにくい]

【出力形式】

📱 **モバイルLP最適化戦略**

**モバイルファーストの重要性**

```
【なぜモバイル最適化が必要か】

・訪問者の60-80%がスマホから
・GoogleはモバイルファーストIndex
・スマホとPCでは行動が違う

【モバイルユーザーの特徴】

・スキマ時間に閲覧
・スクロールは速い
・タップは親指
・通信環境は様々
・集中力は短い
```

**最適化チェックリスト**

```
【ファーストビュー】
━━━━━━━━━━━━━━━━━━━━

□ ヘッドラインが画面内に収まる
□ フォントサイズが読みやすい（16px以上）
□ CTAボタンが見える位置にある
□ 画像が重すぎない
□ 余計な要素を削除

【テキスト・フォント】
━━━━━━━━━━━━━━━━━━━━

□ フォントサイズ16px以上
□ 行間1.5em以上
□ 1行の文字数30-40文字以内
□ 段落間に十分な余白
□ コントラスト比4.5:1以上

【CTAボタン】
━━━━━━━━━━━━━━━━━━━━

□ タップしやすいサイズ（44px以上）
□ 親指が届く位置に配置
□ 周囲に十分な余白
□ 目立つ色・デザイン
□ フローティングCTAを検討

【画像・メディア】
━━━━━━━━━━━━━━━━━━━━

□ 画像を圧縮（WebP形式推奨）
□ 遅延読み込み（Lazy Load）
□ 適切なサイズで配信
□ 動画は自動再生OFF
□ altテキストを設定

【フォーム】
━━━━━━━━━━━━━━━━━━━━

□ 入力項目を最小限に
□ 入力欄は十分な大きさ
□ 適切なキーボードタイプ
□ オートコンプリート対応
□ エラー表示がわかりやすい

【読み込み速度】
━━━━━━━━━━━━━━━━━━━━

□ 3秒以内に表示
□ First Contentful Paint < 2秒
□ 不要なスクリプト削除
□ CSSの最適化
□ CDN利用を検討
```

**モバイル向けレイアウト**

```
【ファーストビューのモバイル設計】

┌─────────────────┐
│    [ロゴ]        │
├─────────────────┤
│                 │
│  ヘッドライン    │
│   （2-3行）     │
│                 │
│  サブヘッド      │
│                 │
│  [CTAボタン]    │

│                 │
└─────────────────┘

要素間：8-16px

【CTAボタンの配置】

■ フローティングCTA（追従型）

│                 │
│                 │

│  [今すぐ申込む]  │ ← 常に画面下部に表示
└─────────────────┘

■ セクション末CTA

各セクションの最後に
CTAボタンを配置

---

【アコーディオン活用】

長いコンテンツは折りたたみ

▼ よくある質問（タップで展開）
▼ 詳細な仕様（タップで展開）
▼ 利用規約（タップで展開）
```

**速度改善施策**

```
【画像最適化】
━━━━━━━━━━━━━━━━━━━━

■ 形式
・WebP形式を使用
・JPEG/PNGより30-50%軽量

■ サイズ
・表示サイズに合わせて配信
・srcset属性でレスポンシブ対応

■ 圧縮
・画質80-85%で十分
・ツール：TinyPNG, Squoosh

---

【コード最適化】
━━━━━━━━━━━━━━━━━━━━

■ CSS
・未使用CSSを削除
・クリティカルCSSをインライン化
・ファイルを圧縮

■ JavaScript
・不要なスクリプト削除
・遅延読み込み（defer/async）
・コード分割

■ HTML
・コメント削除
・空白文字削除

---

【インフラ】
━━━━━━━━━━━━━━━━━━━━

□ CDN利用
□ キャッシュ設定
□ HTTP/2対応
□ Gzip圧縮
□ 高速サーバー

---

【測定ツール】

・Google PageSpeed Insights
・Lighthouse
・GTmetrix
・Web Vitals

目標スコア：
・Performance: 90以上
・LCP: 2.5秒以下
・FID: 100ms以下
・CLS: 0.1以下
```

**モバイル向けコピーライティング**

```
【モバイル向け文章の特徴】

■ 短く、シンプルに
PC：1段落3-5行
モバイル：1段落2-3行

■ 箇条書きを活用
長い文章より
・ポイント1
・ポイント2
・ポイント3

■ 見出しで区切る
スクロールしながら
見出しだけでも理解できるように

■ 結論ファースト
最も重要なことを最初に

---

【例】

PC向け：
「弊社のサービスは、10年以上の実績を持つ
専門チームが、お客様一人ひとりの課題に
合わせたソリューションを提供します。」

モバイル向け：
「10年の実績。
あなた専用のソリューション。」
```

**モバイル特有の注意点**

```
【タップターゲット】

・最小44×44px
・ボタン間に8px以上の余白
・リンクも十分な大きさに

---

【入力フォーム】

・type="email"でメールキーボード
・type="tel"で数字キーボード
・autocomplete属性を活用
・入力欄は大きめに

---

【ポップアップ・モーダル】

・全画面を覆うモーダルは避ける
・閉じるボタンを大きく
・スクロール可能に

---

【横スクロール禁止】

・viewport設定を確認
・画像がはみ出さないように
・表は横スクロール可にするか分割
```
