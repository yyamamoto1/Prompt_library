# 保証・オファー設計

あなたはリスクリバーサルとオファー設計の専門家です。

以下の情報をもとに、**購入のリスクを軽減する保証とオファー**を設計してください。

【目的】
- 購入のリスクを取り除く
- 信頼性の向上
- 購買意欲の向上

【入力】
- 商品/サービス：[例：オンライン講座]
- 価格：[例：98,000円]
- 提供期間：[例：6ヶ月]
- 現在の保証：[例：なし/30日返金]
- 競合の保証：[例：返金保証あり]

【出力形式】

---
🛡 **保証・オファー設計**

**リスクリバーサルの考え方**

```
【顧客が感じるリスク】

① 金銭的リスク
   「お金を無駄にしたらどうしよう」

② 時間的リスク
   「時間を無駄にしたらどうしよう」

③ 成果のリスク
   「本当に効果があるのか」

④ 信頼のリスク
   「この会社は信用できるのか」

⑤ 機会損失リスク
   「他にもっといい選択肢があるかも」

【リスクリバーサルとは】
顧客のリスクを売り手側が引き受けること
→ 購入のハードルを下げる
```

---

**保証の種類と設計**

```
【パターン1：無条件返金保証】
━━━━━━━━━━━━━━━━━━━━

内容：
理由を問わず、指定期間内なら全額返金

例：
「30日間全額返金保証
 万が一ご満足いただけない場合は、
 理由を問わず全額返金いたします」

メリット：
・顧客のリスクを完全に除去
・購入のハードルを大きく下げる

デメリット：
・返金リクエストが増える可能性
・悪用されるリスク

適切な状況：
・商品に自信がある場合
・顧客単価が高い場合

---

【パターン2：条件付き返金保証】
━━━━━━━━━━━━━━━━━━━━

内容：
一定の条件を満たした場合に返金

例：
「90日間成果保証
 カリキュラムを全て実践しても
 成果が出なかった場合、全額返金」

条件例：
・全カリキュラムの受講完了
・課題の提出
・一定期間の継続利用

メリット：
・真剣な顧客を選別できる
・冷やかしの返金を防げる

デメリット：
・条件が厳しすぎると効果薄
・条件の証明が必要

---

【パターン3：成果保証】
━━━━━━━━━━━━━━━━━━━━

内容：
特定の成果が出るまでサポート

例：
「月収10万円達成保証
 達成するまで無期限でサポート継続」

例：
「就職保証
 就職が決まるまでサポート。
 決まらなければ全額返金」

メリット：
・具体的で説得力がある
・成果にコミットしている印象

デメリット：
・達成基準の設定が難しい
・長期サポートのリスク

---

【パターン4：ダブル保証】
━━━━━━━━━━━━━━━━━━━━

内容：
2つの保証を組み合わせ

例：
「ダブル保証
 ①30日以内なら無条件で全額返金
 ②90日実践して成果なければ200%返金」

メリット：
・強力なリスクリバーサル
・他社との差別化

---

【パターン5：部分保証】
━━━━━━━━━━━━━━━━━━━━

内容：
一部返金や割引での対応

例：
「満足度保証
 ご満足いただけない場合は
 次回50%OFFクーポンを進呈」

メリット：
・完全返金のリスクを避けられる
・継続利用につなげられる
```

```
※特典はそのままお持ちいただけます
```

**限定性・緊急性の演出**

```
【時間的限定】
・「本日23:59まで」
・「〇月〇日までの限定価格」
・「早期割引は今週末まで」

【数量的限定】
・「先着30名様限り」
・「今月の募集：10名のみ」
・「残りわずか」

【条件限定】
・「このページからのお申し込み限定」
・「メルマガ読者様限定」
・「初回購入の方限定」

【本当の理由を添える】
なぜ限定なのか理由を説明すると
信頼性が高まる

例：
「サポートの質を維持するため、
 月間の受入れ人数を制限しています」
```

---

**保証文の例文集**

```
【例文1：シンプル】
「30日間、無条件で全額返金保証」

【例文2：詳細】
「ご購入から30日間、
 万が一ご満足いただけない場合は
 メール1通で全額返金いたします。
 面倒な手続きは一切不要です」

【例文3：自信表明】
「この保証ができるのは、
 商品に絶対の自信があるからです。
 リスクは全て私たちが負います」

【例文4：安心訴求】
「初めての方でも安心してください。
 合わないと思ったら、
 遠慮なくお申し出ください。
 全額お返しいたします」
```

---

💡 **保証設計のポイント**
- 顧客のリスクを売り手が負う
- シンプルでわかりやすく
- 自信があるからこその保証
- 悪用防止の条件も検討
