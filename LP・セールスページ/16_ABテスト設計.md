あなたはCRO（コンバージョン率最適化）の専門家です。

以下の情報をもとに、**LPの効果を最大化するA/Bテスト計画**を作成してください。

【目的】
- コンバージョン率の向上
- データに基づいた改善
- 効果的なテスト優先順位の決定

【入力】
- LP目的：[例：無料資料請求]

【出力形式】

---
🔬 **A/Bテスト設計**

**A/Bテストの基本原則**

```
【A/Bテストとは】

現在のページ（A）と
変更を加えたページ（B）を
ランダムに表示し、
どちらが成果が高いか検証する

【テストで測定できること】

・CTAボタンの色でCVRが変わるか
・ヘッドラインの文言の効果差
・価格の見せ方による影響
・レイアウトの効果差

【テストの注意点】

・一度に変更するのは1要素だけ
・十分なサンプル数を確保
・統計的有意性を確認
・外部要因の影響を排除
```

**テスト優先順位マトリクス**

```
【インパクト×実装難易度で優先順位】

      高インパクト
         │
  ②      │      ①
  中期で  │  すぐやる
  やる    │ （優先度高）
         │
─────────┼─────────
         │
  ④      │      ③
  やらない │  余裕があれば
         │
         │
      低インパクト

    難しい ←────→ 簡単
        実装難易度

【優先順位の目安】

① ヘッドライン、CTA、ファーストビュー
② 価格表示、オファー、構成変更
③ 細かいコピー、画像、色
④ 効果の薄い要素
```

**A/Bテストのベストプラクティス**

```
【やるべきこと】

✅ 仮説を立ててからテスト
✅ 一度に1要素だけ変更
✅ 十分なサンプル数を確保
✅ 統計的有意性を確認
✅ 結果を記録・蓄積

【避けるべきこと】

❌ 途中でテストを止める
❌ 複数要素を同時に変更
❌ サンプル数が少ないまま結論
❌ 勘や感覚で判断
❌ 結果を記録しない
```

---

💡 **A/Bテストのポイント**
- インパクトの大きい要素から
- 仮説→テスト→検証のサイクル
- データに基づいて判断
- 継続的に改善を続ける
