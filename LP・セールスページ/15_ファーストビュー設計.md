# ファーストビュー設計

あなたはWebデザインとLP最適化の専門家です。

以下の情報をもとに、**最初の3秒で興味を引くファーストビュー**を設計してください。

【目的】
- 3秒で興味を引く
- ターゲットに「自分のことだ」と認識させる
- 続きを読ませる

【入力】
- 商品/サービス：[例：オンライン英会話]
- ターゲット：[例：海外出張で英語が必要なビジネスマン]
- 主なベネフィット：[例：3ヶ月で実践ビジネス英語習得]
- 実績・権威性：[例：導入企業500社]
- デバイス：[例：スマホ重視/PC重視/両方]

【出力形式】

**ファーストビューの重要性**

```
【3秒ルール】

訪問者は3秒以内に判断する：
・自分に関係あるか
・続きを読む価値があるか
・信頼できそうか

【ファーストビューで伝えること】

① 誰のためのページか
② 何が得られるか
③ なぜ信頼できるか
④ 次に何をすればいいか
```

**ヘッドライン配置例**

```
【ターゲット明示 + ベネフィット】

     〇〇でお悩みの方へ

  たった3ヶ月で
  [成果]を実現する方法

     [導入500社] [満足度98%]

        [無料で始める]

---

【数字 + ベネフィット】

     累計10,000名が実践

  [課題]を[期間]で解決
  [商品名]

     [今すぐ詳細を見る]

---

【問いかけ型】

  まだ[従来の方法]で
  消耗していませんか？

  [新しい方法]なら
  [ベネフィット]が手に入ります

     [詳しく見る]
```

**ビジュアル要素の選び方**

```
【効果的なビジュアル】

■ 人物写真
・ターゲットと同じ属性の人物
・成果を得た後の表情
・使用シーン

■ 商品画像
・商品の実物
・使用イメージ
・Before/After

■ イメージ画像
・理想の状態を表現
・感情を喚起する
・ブランドイメージと一致

【ビジュアル選定のポイント】

✅ ターゲットが共感できる
✅ ベネフィットを視覚化
✅ 高品質（ぼやけ・低解像度NG）
✅ ブランドトーンと一致
✅ モバイルでも認識可能
```

**ファーストビューチェックリスト**

```
【必須チェック項目】

□ 3秒で何のページかわかる
□ ターゲットが自分事と感じる
□ ベネフィットが明確
□ 続きを読みたくなる
□ 信頼性を感じる

【デザインチェック】

□ 文字が読みやすい
□ 余白が適切
□ モバイルで崩れない
□ 読み込み速度が速い
□ CTAが目立つ
```
